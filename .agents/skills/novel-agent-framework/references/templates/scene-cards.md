# Scene Cards Template

## Required Artifact Path

- `docs/novel-projects/<novel-id>/artifacts/scene-cards.md`

## Global Header

At the top of the file, record:

- project identifier
- act or section mapping
- chapter-count basis
- POV roster
- scene-card version

## Required Per-Chapter Sections

For each chapter, include:

- chapter identifier
- chapter purpose
- chapter market function
- opening state
- closing state
- chapter hook
- scene list

## Required Per-Scene Fields

Each scene must include:

- scene identifier
- chapter identifier
- POV character
- timeline position
- location
- scene goal
- conflict source
- stakes
- obstacle
- information revealed
- exposition load
- emotional beat
- value shift
- disaster / reveal / reversal
- outcome
- sequel function
- carry-over to next scene
- continuity watchouts

## Field Guidance

### `chapter market function`

State what this chapter is doing for the target format, such as:

- cold-open hook
- escalation beat
- bonding beat
- twist delivery
- revelation bridge
- pre-climax tightening
- emotional recovery

### `value shift`

Describe the before/after change in dramatic value, such as:

- trust -> suspicion
- safety -> danger
- ignorance -> knowledge
- hope -> despair

### `disaster / reveal / reversal`

At least one of these should be materially present in a dramatic scene.
If none is present, explain why the scene still deserves to exist.

### `sequel function`

Describe what the aftermath of the scene enables:

- decision
- regrouping
- emotional processing
- tactical shift
- new objective

### `continuity watchouts`

Record anything later drafting or proofreading must keep consistent:

- injury state
- object possession
- time elapsed
- weather / environment
- relationship temperature
