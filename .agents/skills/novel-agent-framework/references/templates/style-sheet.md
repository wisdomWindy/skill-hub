# Style Sheet Template

## Required Artifact Path

- `docs/novel-projects/<novel-id>/artifacts/style-sheet.md`

## Required Sections

- project identifier
- language and script
- narration mode
- tense policy
- POV policy
- diction register
- title and heading conventions
- naming conventions
- honorific and address conventions
- place-name conventions
- worldbuilding term conventions
- formatting conventions
- number and date conventions
- punctuation conventions
- dialogue formatting rules
- emphasis rules
- taboo words and restricted usage
- continuity-sensitive attributes
- update log

## Section Guidance

### `narration mode`

State explicitly:

- first person / close third / omniscient / hybrid
- if hybrid, where transitions are allowed

### `POV policy`

Describe:

- who may hold POV
- whether chapter-level POV switches are allowed
- whether scene-level POV switches are allowed
- what counts as POV leakage

### `diction register`

Define the intended prose register:

- plain / elevated / lyrical / commercial-fast / literary-minimal / etc.
- taboo overuse patterns to avoid
- desired sentence rhythm tendency

### `naming conventions`

Define rules for:

- person names
- aliases
- titles
- capitalization
- transliteration

### `continuity-sensitive attributes`

Track details that proofread and continuity review must verify, such as:

- eye color
- scars
- injuries
- ranks
- location names
- object names
- timeline anchors

### `update log`

Every meaningful change to the style sheet should be logged with:

- date or iteration
- changed item
- reason
- downstream impact
