# Novel Agent Framework Overview

This framework exists to help Codex behave like a disciplined fiction production workflow rather than a one-shot text generator.

## Core idea

The workflow assumes:

- a rough story idea is not yet a publishable novel,
- the novel should be built through explicit stages,
- review stages must be able to force rollback,
- and completion means acceptance, not merely draft existence.

## Default deliverables

The framework aims to create and maintain:

- market and comparable-title positioning,
- platform and format constraints,
- premise and market framing,
- style sheet,
- character bible,
- structural outline,
- scene cards,
- chapter plan,
- manuscript files,
- continuity ledger,
- revision log,
- beta feedback report,
- accepted risk register,
- copyedit log,
- proofread log,
- acceptance report.

## Default strategy for long-form work

For long-form fiction, prefer staged convergence:

1. lock the premise,
2. lock the market lane and reader promise,
3. lock the architecture,
4. lock the character logic,
5. lock the chapter and scene plan,
6. draft in batches,
7. review for continuity and structure,
8. gather beta-style feedback,
9. polish at sentence level,
10. copyedit for consistency and style,
11. proofread for final surface defects,
12. review against the acceptance standard.

## Main control principle

The main skill decides:

- what stage is active,
- whether the current output is sufficient,
- whether to advance or roll back,
- and whether the project is truly complete.

Subskills should stay narrow and stage-local.
