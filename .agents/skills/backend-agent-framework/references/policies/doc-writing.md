# Documentation Writing Policy

## Scope

This policy defines how request artifacts should be written so they can be executed and reviewed reliably.

## Standards

- Write for execution clarity, not prose elegance.
- Make scope, assumptions, risks, and non-goals explicit.
- Avoid vague acceptance language such as "works correctly" without observable criteria.
- Make plan and verification artifacts point back to concrete spec items.
- Keep clarifications and decisions in repository artifacts rather than only in chat history.

## Non-Goals

- This policy does not define request lifecycle gates.
- This policy does not replace artifact templates.
