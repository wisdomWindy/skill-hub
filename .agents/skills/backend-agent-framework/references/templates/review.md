# 评审模板

## 必需工件路径

- `docs/requests/<request-id>/modules/<module-id>/review/review.md`

## `modules/<module-id>/review/review.md`

必需章节：

- 模块标识与范围
- 阻塞问题
- 非阻塞问题
- 已接受风险
- 后续事项
- 整洁代码评估
- 设计模式评估
- 安全评估
- 代码上下文结构评估
- 合并准备度总结

`整洁代码评估` 必须包含：

- 结果（`通过` 或 `失败`）
- 关键发现
- 失败时必须跟进的动作

`设计模式评估` 必须包含：

- 结果（`通过` 或 `失败`）
- 关键发现
- 失败时必须跟进的动作

`安全评估` 必须包含：

- 结果（`通过`、`失败` 或 `不适用`）
- 关键发现
- 失败时必须跟进的动作
