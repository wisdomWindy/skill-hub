# 背景与目标

当前 SkillHub 页面混有英文展示文案和乱码中文。目标是将所有页面可见文本统一为中文，并保持现有页面功能和数据契约不变。

# 范围内

- 首页、技能列表页、技能详情页文本。
- 页眉、页脚、主题切换、分页、空状态、安装命令、相关技能、版本记录组件文本。
- `_data/config.yaml` 站点描述和分类标签。
- `_data/skills/*.yaml` 中面向页面展示的名称、描述、标签、变更记录和示例标题。
- 测试中与中文分类标签相关的期望值。

# 范围外

- 不重构布局、样式和交互。
- 不改路由、查询参数、分类 key、skill id、status 枚举。
- 不翻译安装命令和代码块中的 CLI 参数。
- 不新增 i18n 框架。

# 触发与开始条件

用户直接要求全站页面文本改成中文。当前指令视为 spec 和 plan 的执行授权。

# 需求拆分摘要

单模块：全站可见文本中文化。

# 用户流程

用户进入首页、列表页或详情页时，页面静态文案、空状态、反馈信息和示例技能内容均以中文展示；搜索、筛选、排序、分页和复制命令行为保持原样。

# 页面与模块设计

不改页面结构。仅替换文本内容和 YAML 展示字段。

# 功能完整行为拆解

- 首页：标题、副标题、搜索占位、按钮、分类入口、性能说明、最新技能区和空状态均为中文。
- 列表页：标题、副标题、搜索占位、排序选项、分类侧栏、结果摘要、空状态和重置按钮均为中文。
- 详情页：面包屑、fallback 详情、详情区标题、示例区标题、缺失状态、返回入口均为中文。
- 组件：复制命令反馈、安装命令标题、安装量、更新时间、相关技能、版本记录、分页按钮、主题切换和页脚均为中文。
- 数据：示例技能面向用户的字段均为中文，机器字段保持不变。

# 设计约束

- 保持现有组件职责边界，不引入新抽象。
- 不使用全局替换破坏代码标识符。
- 中文文本应可读、自然，修复已有乱码。

# 项目脚手架与初始化决策

非 greenfield，不涉及脚手架。

# 变化轴与模式决策

- 变化轴：展示文案语言。
- 模式决策：不引入 i18n；本次范围是单语言中文化，直接替换静态文案更小更清晰。

# 代码上下文与影响假设

见 `artifacts/code-context.md`。当前变更主要影响 Vue 模板和 YAML 展示数据。

# API 与数据契约

无后端 API。YAML 数据契约字段名不变，字段值中面向用户的文本中文化。

# 上下文与依赖来源

- `tsconfig.app.json`
- `tsconfig.json`
- `src/env.d.ts`
- `src/types/content.ts`
- `src/types/skill.ts`

# 边界情况

- 搜索仍应能匹配中文名称和中文描述。
- 英文 CLI 命令、参数、URL、版本号保持不变。
- 空状态和缺失详情页不能残留英文。

# 验收标准

- 全站页面可见静态文本为中文，品牌名和技术命令除外。
- 现有乱码中文被替换为正确中文。
- `npm run test` 通过。
- `npm run typecheck` 通过。
- `npm run build` 通过。

# 人工评审与交接

交付时列出修改范围和验证结果。

# 风险

- 示例技能名称中文化可能影响依赖英文查询的测试或手动搜索预期，需同步测试或保留必要英文查询样例。
