# Page Design

## Delivery Unit Identifier

- Request: `prd-skillhub-personal-skill-distribution`
- Module: `module-02-public-discovery`

## Page Objective

让访客在进入 SkillHub 后，第一眼就能快速理解站点内容、筛选维度和技能分发价值，并在首页与列表页之间顺滑切换完成发现路径。

## Target Users or Scenarios

- 第一次访问站点、需要快速感知“这里有哪些技能”的访客
- 已知大致方向、需要靠分类或搜索缩小范围的访客
- 需要按发布时间或名称浏览技能列表的访客

## Layout Structure

- 首页：
  - 顶部延续全局壳层头部
  - 首屏 Hero 区：大标题、简短说明、搜索框、主行动按钮
  - 分类快捷入口区：横向滚动标签
  - 最新技能卡片区：4 列响应式网格
- 列表页：
  - 顶部信息区：页面标题、结果数、搜索框、排序控件
  - 主体双栏区：
    - 左侧分类侧边栏
    - 右侧技能卡片网格
  - 底部分页区

## Visual Hierarchy

- 首页 Hero 的搜索框与“浏览技能”主行动是第一视觉焦点
- 分类快捷入口作为第二焦点，强化信息架构
- 技能卡片保持信息密度统一：名称 > 描述 > 分类 / 版本
- 列表页优先强调“筛选路径”和“结果上下文”，而不是首页式品牌宣言

## Section Breakdown

- 首页 Hero：
  - 大标题
  - 站点价值说明
  - 可输入的搜索框
  - 通往列表页的按钮
- 分类快捷区：
  - 全部分类入口
  - 分类标签滚动条
- 最新技能区：
  - 标题
  - 技能卡片网格
- 列表页筛选栏：
  - 分类列表
  - 当前选中状态
- 列表页结果区：
  - 结果总数
  - 排序控件
  - 卡片网格
  - 空状态卡片
  - 分页导航

## Styling Direction

- 延续基础模块的“深色玻璃卡片 + 薄荷绿强调色”方向
- 首页 Hero 使用更大胆的排版与更宽松的留白
- 列表页用稍收敛的结构化布局，突出信息效率
- 分类标签和排序控件保持圆角胶囊风格，避免通用后台感

## Interaction Skeleton

- 首页搜索框输入后：
  - 以实时客户端过滤为目标
  - 可直接跳转到列表页并带 query
- 分类快捷入口：
  - 点击后进入列表页并带 category
- 列表页搜索、分类、排序可叠加
- 分页采用页码切换作为主路径
- 无结果时显示明确空状态提示与重置入口

## Responsive Behavior

- 桌面：
  - 首页技能区 4 列
  - 列表页双栏布局
- 平板：
  - 首页 2 列或 3 列
  - 列表页侧栏压缩但仍可见
- 移动：
  - 首页单列 Hero + 单列卡片流
  - 列表页分类侧栏折叠为顶部横向分类条
  - 搜索、排序、结果统计垂直堆叠

## Design Risks

- 若首页和列表页都过度承载筛选逻辑，信息层级会混乱
- 若移动端仍强保双栏，会严重压缩卡片可读性
- 若分类入口过多且没有视觉优先级，首页会碎片化

## Open UI Questions

- 当前决定以“页码分页”为主路径，更适合静态与可预测列表；滚动加载保留为潜在扩展，不进入本轮 spec
