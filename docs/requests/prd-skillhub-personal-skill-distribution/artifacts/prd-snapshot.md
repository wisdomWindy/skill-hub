# PRD Snapshot

## Source Summary

- 来源文件：[PRD.md](/Users/staff/Desktop/testProject/skill-hub/PRD.md)
- 产品：SkillHub，个人技能分发平台
- 发布模式：单维护者分发模型
- 技术栈方向：Vue 3 + Vite + Pinia + GitHub Pages
- 用户最新澄清：本次实现不包含后台管理，仅交付根据 GitHub 仓库部署的纯静态 GitHub Pages 站点
- 产品本质：公开只读的技能展示与安装分发站点

## Key Business Goals

- `G1`：为每一个技能提供结构化展示页与完整文档
- `G2`：让用户通过搜索和分类快速找到技能
- `G3`：提供清晰、可一键复制的终端安装命令
- `G4`：维护者仅更新数据文件即可发布新技能，降低维护成本
- `G5`：可选追踪技能使用热度

## Explicit Behavior Constraints

### 公开技能浏览

- 首页展示所有已上架技能卡片，包含图标、名称、简短描述、分类标签、当前版本号。
- 顶部导航提供分类筛选，默认分类包括：开发工具、数据处理、DevOps、安全、办公自动化、其他。
- 全局搜索框按技能名称与描述做模糊检索，实时显示匹配结果。
- 排序支持发布时间新到旧、名称 A 到 Z。
- 分页每页 20 个技能，支持页码跳转或滚动加载。
- 空状态需要给出友好提示。
- 未登录访客可完整访问所有公开页面。
- 搜索响应目标：客户端预索引下小于 500ms；非功能目标写为小于 300ms。
- 分类筛选与搜索条件可叠加。

### 技能详情页

- 详情页展示大图标、名称、版本、分类、作者、最后更新时间。
- 详细描述支持 Markdown 完整渲染，包含功能说明、使用场景、前置依赖、环境要求。
- 安装命令区域必须高亮展示完整命令，带一键复制按钮与成功 Toast。
- 参数配置说明列出环境变量和命令行参数。
- 使用示例展示 1 到 3 个典型场景代码块，并做语法高亮。
- 版本历史按时间倒序展示。
- 可选展示安装量计数与同分类相关技能推荐。
- 需要自动生成 Open Graph 元信息。
- 详情页加载目标小于 800ms，PRD 非功能目标中的 LCP 要求小于 1.5s。

### 数据存储与部署

- 技能数据以 `_data/skills/{id}.yaml` 存储，每个技能一个文件。
- 站点配置位于 `_data/config.yaml`。
- 内容维护发生在 GitHub 仓库内，站点只负责消费仓库中的 YAML 数据并静态构建。
- GitHub Actions 监听 `main` push 并构建部署到 `gh-pages`。
- GitHub Pages 对外托管站点。

### 非功能与安全约束

- 首页首屏加载小于 1.2s，详情页 LCP 小于 1.5s。
- 构建失败时保留上一次成功构建。
- 全站 HTTPS。
- GitHub Token 不得暴露给前端静态资源。
- Markdown 渲染需要 XSS 过滤。
- 默认暗色主题，支持手动切换亮色，并跟随系统偏好。
- 响应式适配桌面、平板、移动端。

## Forms, Tables, Displays, and Interactions Extracted From Source

### 公开站点

- 首页：大标题、搜索框、分类快捷入口、最新技能卡片网格。
- 列表页：左侧可折叠分类侧边栏，右侧卡片网格，顶部排序下拉与结果数量统计。
- 全局组件：Logo、站点名、搜索入口、主题切换按钮。

### 详情页

- 左侧主区：面包屑、技能名称、版本、分类标签、Markdown 描述、使用示例。
- 右侧边栏：安装命令高亮卡片、复制按钮、版本历史、安装量计数。
- 详情页还需支持相关技能推荐与页面分享元信息。

## Workflow and State Rules Extracted From Source

- 维护者在 GitHub 仓库内维护技能数据与站点代码，推送到 GitHub 后由 GitHub Actions 自动构建部署。
- 访客浏览站点、查看详情、复制安装命令，在终端执行安装。
- 技能状态只有 `published` / `archived`；下架后公开页面不可见，但已安装用户不受影响。
- 公开页面在 JavaScript 未完全加载时仍应可读。
- 搜索和筛选作用于客户端内存中的技能列表。
- GitHub 仓库是内容与代码的事实源，变更可追溯和回滚。
